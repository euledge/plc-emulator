# PLCEmulator 機能要求一覧

## FR-001: MCプロトコル / SLMP 通信エミュレーション

### FR-001-1: フレームフォーマット対応
- [ ] 1Eフレーム（バイナリ）の受信解析・応答生成
- [ ] 3Eフレーム（バイナリ）の受信解析・応答生成
- [ ] 3Eフレーム（ASCII）の受信解析・応答生成
- [ ] 4Eフレーム（バイナリ）の受信解析・応答生成
- [ ] SLMP拡張デバイス指定（4バイトアドレス + 2バイトコード）対応

### FR-001-2: コマンド対応
- [ ] `0401` 一括読出し（ワード/ビット）
- [ ] `1401` 一括書込み（ワード/ビット）
- [ ] `0403` ランダム読出し（ワード + ダブルワード混在）
- [ ] `1402` ランダム書込み（ワード + ダブルワード混在）
- [ ] `0801` モニタ登録
- [ ] `0802` モニタ実行
- [ ] `0101` CPU型名読出し
- [ ] `1001` リモートRUN
- [ ] `1002` リモートSTOP
- [ ] `0619` ループバックテスト
- [ ] `1630` リモートパスワードアンロック
- [ ] `1631` リモートパスワードロック
- [ ] 未サポートコマンドに対するエラーコード応答（0xC059）

### FR-001-3: デバイス対応

#### ビットデバイス
- [ ] X（入力リレー, コード: 0x9C, 16進数アドレス）
- [ ] Y（出力リレー, コード: 0x9D, 16進数アドレス）
- [ ] M（内部リレー, コード: 0x90, 10進数アドレス）
- [ ] L（ラッチリレー, コード: 0x92, 10進数アドレス）
- [ ] F（アナンシエータ, コード: 0x93, 10進数アドレス）
- [ ] V（エッジリレー, コード: 0x94, 10進数アドレス）
- [ ] B（リンクリレー, コード: 0xA0, 16進数アドレス）
- [ ] SM（特殊リレー, コード: 0x91, 10進数アドレス）
- [ ] SB（リンク特殊リレー, コード: 0xA1, 16進数アドレス）
- [ ] S（ステップリレー, コード: 0x98, 10進数アドレス）
- [ ] TS（タイマ接点, コード: 0xC1, 10進数アドレス）
- [ ] TC（タイマコイル, コード: 0xC0, 10進数アドレス）
- [ ] CS（カウンタ接点, コード: 0xC4, 10進数アドレス）
- [ ] CC（カウンタコイル, コード: 0xC3, 10進数アドレス）

#### ワードデバイス
- [ ] D（データレジスタ, コード: 0xA8, 10進数アドレス）
- [ ] W（リンクレジスタ, コード: 0xB4, 16進数アドレス）
- [ ] R（ファイルレジスタ, コード: 0xAF, 10進数アドレス）
- [ ] ZR（拡張ファイルレジスタ, コード: 0xB0, 16進数アドレス）
- [ ] SD（特殊レジスタ, コード: 0xA9, 10進数アドレス）
- [ ] SW（リンク特殊レジスタ, コード: 0xB5, 16進数アドレス）
- [ ] TN（タイマ現在値, コード: 0xC2, 10進数アドレス）
- [ ] CN（カウンタ現在値, コード: 0xC5, 10進数アドレス）

### FR-001-4: エミュレート対象PLC機種
- [ ] Q03UDE（MELSEC-Q）
- [ ] Q06UDE（MELSEC-Q）
- [ ] R04CPU（MELSEC iQ-R）
- [ ] R08CPU（MELSEC iQ-R）
- [ ] FX5U（MELSEC iQ-F）
- [ ] L06CPU（MELSEC-L）

### FR-001-5: エラーコード応答
- [ ] 0x0000 正常完了
- [ ] 0xC050 コマンド/応答タイプ仕様外
- [ ] 0xC051 開始アドレス範囲外
- [ ] 0xC056 デバイス指定エラー
- [ ] 0xC058 デバイスアドレス無効
- [ ] 0xC059 未サポートコマンド
- [ ] 0xC05B パラメータエラー
- [ ] 0xC061 データ長不一致
- [ ] エラー応答の手動ON/OFF切替

## FR-002: 通信サーバ

### FR-002-1: 接続方式
- [ ] TCP接続対応
- [ ] UDP接続対応
- [ ] TCP/UDP選択可能
- [ ] 1クライアント接続仕様
- [ ] ポート番号設定可能（1024-65535、デフォルト: 5000）

### FR-002-2: 接続管理
- [ ] クライアント切断検知（recv=0検出）
- [ ] 切断後の自動クリーンアップ
- [ ] 再接続対応（サーバ再起動不要）

---

## FR-003: レイテンシエミュレーション

- [ ] レイテンシ無し（即座に応答）
- [ ] 固定遅延（指定ミリ秒後に応答）
- [ ] ランダム遅延（範囲指定）
- [ ] 正規分布遅延（平均・標準偏差指定）
- [ ] タイムアウト模擬（確率指定で応答しない）
- [ ] レイテンシはMCプロトコル/SLMP応答にのみ適用（Web UIには適用しない）
- [ ] 遅延統計表示（平均・最大・最小）

---

## FR-004: デバイスメモリ管理

### FR-004-1: 基本機能
- [ ] デバイス値の読み書き
- [ ] ビットデバイスの16ビットワード内パッキング
- [ ] PLC機種に応じたデバイスレンジチェック

### FR-004-2: Web UIからの値設定
- [ ] 画面上でデバイス値を直接編集（設定した値を通信で使用）
- [ ] デバイス値の一括クリア

### FR-004-3: 永続化
- [ ] デバイス値のJSON形式での保存
- [ ] デバイス値のJSON形式での読込（復元）

---

## FR-005: デバイスメモリスクリプトエンジン

### FR-005-1: DSL仕様
- [ ] YAMLベースのスクリプト定義
- [ ] 周期実行（periodic）: 一定間隔でデバイス値を更新
- [ ] 条件実行（conditional）: デバイス値に基づく分岐
- [ ] シーケンス実行（sequence）: ステップ順次実行
- [ ] ランプ（ramp）: 値の段階的変化

### FR-005-2: 式評価
- [ ] 数学関数: sin, cos, abs, min, max, floor, ceil, int
- [ ] 範囲制限: clamp(x, lo, hi)
- [ ] 乱数: random(), randint(a, b)
- [ ] 波形生成: square(t, period), triangle(t, period), sawtooth(t, period)
- [ ] デバイス値参照（式内で D100, M0 等を変数として使用）
- [ ] 組み込み変数: t（経過秒）, dt（前回からの経過）, tick（実行回数）

### FR-005-3: 安全性
- [ ] ast.parseベースのサンドボックス式評価（eval/exec不使用）
- [ ] ファイルシステム・ネットワークアクセス不可
- [ ] import文不可
- [ ] 評価タイムアウト

### FR-005-4: スクリプト管理
- [ ] スクリプトの開始/停止/一時停止
- [ ] プリセットシナリオ（テンプレート提供）
- [ ] バリデーション（YAML構文 + 式の安全性チェック）

---

## FR-006: Web UI

### FR-006-1: サーバ設定画面
- [ ] プロトコル選択: MCプロトコル 3E / 1E / 4E / SLMP
- [ ] 通信方式選択: TCP / UDP
- [ ] ポート番号入力
- [ ] データ形式選択: Binary / ASCII
- [ ] PLC機種選択
- [ ] エラー応答 ON/OFF
- [ ] レイテンシ設定

### FR-006-2: デバイスモニタ
- [ ] デバイス種別タブ切替（D, M, X, Y, W, R等）
- [ ] 開始アドレス指定
- [ ] 表示形式切替: DEC符号なし / DEC符号付き / HEX / BIN / FLOAT / ASCII
- [ ] ダブルクリックで値を直接編集
- [ ] WebSocketによるリアルタイム更新

### FR-006-3: 通信ログ
- [ ] タイムスタンプ + 方向（TX/RX）+ HEXダンプ
- [ ] 解析済みコマンド名表示
- [ ] 自動スクロール ON/OFF
- [ ] ログクリア・テキスト保存

### FR-006-4: スクリプトエディタ
- [ ] YAMLテキストエディタ（シンタックスハイライト付き）
- [ ] スクリプト開始/停止/一時停止ボタン
- [ ] 実行中ステータス表示
- [ ] プリセットテンプレート選択
- [ ] バリデーション結果表示

---

## NFR-001: 非機能要求

### NFR-001-1: クロスプラットフォーム
- [ ] Windows, macOS, Linux で動作
- [ ] 特別な開発環境不要（Python + ブラウザのみ）

### NFR-001-2: パフォーマンス
- [ ] ベースレスポンスタイム 50ms 以内（レイテンシエミュレーション無効時）
- [ ] デバイスメモリアクセス O(1)
- [ ] スクリプトエンジンがプロトコル応答をブロックしない

### NFR-001-3: 国際化（i18n）
- [ ] 日本語対応
- [ ] 英語対応
- [ ] JSONベースの翻訳ファイル
- [ ] 動的言語切替

### NFR-001-4: マニュアル忠実性
- [ ] 三菱電機公式マニュアル（SH-080008等）に準拠したプロトコル実装
- [ ] デバイスコード、コマンドコード、エラーコードは公式仕様に忠実
