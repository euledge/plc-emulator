#### 10:31 - リサーチ結果到着（参考ツール分析）

**主要な発見:**
- DummyPLC は C# / .NET 製、Visual Studio 2022 で開発
- 対応: 三菱MCプロトコル/SLMP, OMRON FINS, KEYENCE上位リンク/MCプロトコル, 横河パソコンリンク
- 共有メモリでデバイス値管理（最大4台分）
- ソースコードは有償（BASE.shopで販売）
- 既知の問題: TCP切断後の再接続不可、4Eフレーム未対応、SLMP拡張デバイス指定一部未対応
- GitHubのOSSエミュレータは極めて少ない（ほとんどがクライアント実装）

---

#### 10:33 - リサーチ結果到着（三菱PLCプロトコル調査）

**主要な発見:**
- フレーム形式: 1E（レガシー）、3E（標準）、4E（拡張）
- コマンド: 0401一括読出し, 1401一括書込み, 0403ランダム読出し, 1402ランダム書込み, 0101 CPU型名読出し, 0619ループバックテスト等
- デバイスコード: ビット（X:0x9C, Y:0x9D, M:0x90等）、ワード（D:0xA8, W:0xB4, R:0xAF等）
- MCプロトコルとSLMPの差分: デバイス指定バイト数が異なる（MC:4バイト, SLMP:6バイト）
- エラーコード: 0x0000正常, 0xC059未サポートコマンド等
- 参考マニュアル: SH-080008等

#### 10:41 - ユーザーフィードバック（v1に対して）

**ユーザー回答:**
1. **Windowsの開発環境を持っていない。Windows以外のOSでも実行できるようにしたい** → WPFは不可
2. 初期リリースはMCプロトコル/SLMPでOK
3. Q1: UIは多言語翻訳対応にしたい（最初は日英）
4. Q2: 1クライアントでよい（TCP or UDP選択可能に）
5. Q3: デバイス値の永続化はほしい（画面上で値設定、その値を通信に使用）

**決定事項:**
- 技術スタックを C#/WPF → **Python + Web UI** に変更
- クロスプラットフォーム対応
- i18n対応（日/英）
- 1クライアント接続仕様
