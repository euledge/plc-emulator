# ADR-003: UI アーキテクチャ

## ステータス
**承認済み** (2026-06-20)

## コンテキスト
エミュレータにはデバイスモニタ、通信ログ、設定画面、スクリプトエディタ等のUIが必要である。  
クロスプラットフォームで動作する必要があるため、ネイティブGUIフレームワークは不適切。

## 決定
**ブラウザベースのWeb UI**（FastAPIサーバ + HTML/CSS/JS）を採用する。

## 理由
1. **OS依存なし**: ブラウザはあらゆるOSで利用可能
2. **リアルタイム更新**: WebSocketでデバイス値の変更や通信ログをリアルタイムにプッシュ可能
3. **追加インストール不要**: ブラウザは既にインストールされている
4. **開発効率**: HTML/CSS/JSはUIの表現力が高く、開発ツールも豊富

## UI設計方針
- SPA（Single Page Application）構成
- ダークモード対応の産業用ソフトウェアらしいデザイン
- 多言語対応（日本語/英語、JSONベース翻訳）
- レスポンシブレイアウト

## 通信方式
- **REST API**: 設定変更、デバイス値操作等のリクエスト/レスポンス型通信
- **WebSocket**: デバイス値変更通知、通信ログストリーミング等のリアルタイム通信
