# ADR-004: レイテンシエミュレーション

## ステータス
**承認済み** (2026-06-20)

## コンテキスト
実際のPLC通信では、ネットワーク遅延やCPU処理時間による応答遅延が発生する。  
上位アプリケーション（SCADA/HMI）のタイムアウト処理等をテストするため、応答遅延をエミュレートする機能が必要。

## 決定
MCプロトコル/SLMP応答にのみレイテンシエミュレーションを適用する。  
Web UI（REST API / WebSocket）には一切適用しない。

## 適用範囲

| 通信経路 | レイテンシ適用 |
|:--|:--|
| MCプロトコル/SLMP応答 → SCADA/HMI | **適用する** |
| Web API / WebSocket → ブラウザUI | **適用しない** |

## 遅延モード
| モード | 説明 |
|:--|:--|
| なし | 即座に応答（デフォルト） |
| 固定遅延 | 一定ミリ秒後に応答 |
| ランダム遅延 | 指定範囲内のランダム遅延 |
| 正規分布遅延 | 平均・標準偏差に基づく遅延 |
| タイムアウト模擬 | 一定確率で応答しない |

## 理由
- レイテンシは通信テストのためにPLCプロトコル応答にのみ意味がある
- Web UIは常に高速応答し、ユーザー操作性を損なわない
- `asyncio.sleep()` による非同期遅延でイベントループをブロックしない
